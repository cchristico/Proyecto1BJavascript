IIFEs ahora Blocks

En JavaScript, cada función , cuando se invoca, crea un nuevo contexto de ejecución. Dado que las variables y funciones definidas dentro de una función sólo se puede acceder a su interior, pero no fuera, de ese contexto. 

    // Esta duncion retorna otra funcion que tiene aceso a la varibla privada i
//la funcion retornada es, efectivamente, "privilegiada".

function contador() {
  // `i`  es accedida solo dentro de 'contador'.
  var i = 0;

  return function() {
    console.log( ++i );
  };
}

//Note que counter1 y counter 2 tiene su propio alcance de i.

var counter = contador();
counter(); // logs: 1
counter(); // logs: 2

var counter2 = contador();
counter2(); // logs: 1
counter2(); // logs: 2

i; // ReferenceError: i no esta definida(solo existe dentro de contador()

Immediately Invoked Function Expressions permite encerar los valores para que sean usados dentro de su alcance. En ES6, ahora se tiene la opcion de crear bloques o funciones. 

   //Bloques
{
    //Ayuda a realizar operaciones, en el alncace definido entre {}
    let i = 4;
    console.log(i);
};
