var vs let vs const


var

Javascript solo tenía una forma de declarar variables. Se hacía con la palabra clave "var" seguida del nombre de variable a declarar (o variables separadas por comas). Ejemplo: 

    var variable;

Esta variable podía ser de ámbito global o local. Su ámbito depende del lugar donde se declara pero las posiblidades eran bien limitadas. Para hacer una variable global se tenía que declarar simplemente fuera del cuerpo de una función. En cambio, las variables locales eran declaradas dentro de una función y solo eran accesibles dentro del código de esa función. 

    function () {
var miVar;
  console.log(miVar);
  if (true) {
    miVar = "Hola mundo";
  }
  console.log(miVar);
};

La salida de la consola sería en este caso: 

    undefined
    Hola mundo

Como se puede observar, en cuanto ve una variable ya sea parte de un if o de la propia función pasa a declararla antes de seguir ejecutando el código (Esto es debido a la doble pasada que hace por el código el intérprete). Por eso, en el primer console.log sale como declarada pero sin valor definido.

let

Para solucionar todo esto en ECMAscript ha implementado una forma nueva de declarar variables que ahora será la más indicada en la mayoría de los casos. Se trata de la construcción "let" que veremos mejor con un ejemplo similar: 

    function () {
  console.log(miVar);
  if (true) {
    let miVar = "Hola mundo";
  }
  console.log(miVar);
};

La clave de let es que restringe su existencia al ámbito donde ha sido declarada (cualquier ámbito expresado con unas llaves). Por tanto, la salida de la consola sería en este caso: 

    undefined
    undefined

Esta salida se debe a que let es la nueva forma de declarar variables de bloque, sólo existirán dentro del bloque que las contenga y no contaminará nunca el código a su alrededor.

const

Crea una variable donde su valor es inalterable a lo largo del código. Ejemplo; 

         const MIVARIABLE = 1;
         console. log(MIVARIABLE);

El resultado que devuelve es 1. Pero si a continuacion se vuelve a asigar otro valor a la variable devolvera error.
